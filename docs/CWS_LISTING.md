# Chrome Web Store listing

Use this when submitting or editing the Bunker46 extension in the [Chrome Web Store Developer Dashboard](https://chrome.google.com/webstore/devconsole).

---

## Verantwoording voor tabs (permission justification)

When the store asks why the extension needs the **tabs** permission, you can use:

**Nederlands (Dutch):**

```
De extensie gebruikt het "tabs"-recht om: (1) in het popupvenster het domein van de actieve tab te tonen op het tabblad Machtigingen, zodat u de huidige site aan de whitelist kunt toevoegen of verwijderen; (2) het badge-cijfer op het extensiepictogram bij te werken (aantal verleende machtigingen voor de site in die tab); (3) links te openen in een nieuw tabblad (bijv. naar uw Bunker46-instance bij nostrconnect-koppelingen). Er worden geen tab-URL's of -gegevens naar externe servers gestuurd.
```

**English (for reference):**

The extension uses the "tabs" permission to: (1) show the active tab’s domain in the popup Permissions tab so you can add or remove the current site from the whitelist; (2) update the extension icon badge (number of granted permissions for that tab’s site); (3) open links in a new tab (e.g. to your Bunker46 instance for nostrconnect links). No tab URLs or tab data are sent to external servers.
